# 4/26/2026 4 AM - PCB Design

_Time spent: 8h_

Before touching the PCB layout I reviewed the schematic and found two bugs that would have made the board non-functional. All four MOSFETs had Gate and Drain swapped, the GATE_CHx nets were on the Drain pins, meaning the ESP32 would have driven the wrong terminal entirely. Also had GATE_CH4 on IO14 instead of IO32, and IO14 is a strapping pin. Fixed both before doing anything else.
The ESP32 footprint took about two hours. I was using the WROOM-32 module symbol which numbers pins by the chip's castellated pads, not the dev board header row. Any header footprint assigned to it would route completely wrong nets to wrong holes. Was spending too much time trying to find a replacement so I built a custom symbol and footprint from scratch two rows of 15 pins, 2.54mm pitch, 25.4mm row spacing.
I wanted to add charging to the system so I added TP4056 charging circuits per channel, a dedicated USB-C power input separate from the programming port, and then I also decided to add a rotary encoder for standalone OLED navigation without needing WiFi.
FreeRouting ran three times total. First had display artifacts showing false unrouted connections. Second had 34 unrouted because the ESP32 was placed flipped. Third resolved clean after 21+ passes.
Biggest DRC issue was TP4056 EPAD thermal vias defaulting to 0.2mm drill, JLCPCB minimum is 0.3mm. The scripting console kept throwing errors on multiline code so I fixed all 24 vias by hand across four footprints. Parts matching on JLCPCB also caught LED footprint mismatches before ordering, auto-matched 0603 parts to 0402 pads.
 Also wrote the firmware for the encoder, rotate to switch between channels on the OLED, press to open a menu, long press to go back. From the menu you can start a test, stop it, or set the cutoff voltage per channel.

Once I had a charged cell the discharge loop ran correctly; voltage holding in the flat plateau, capacity counting up steadily.

# 3/5/2026 9 AM - Background research

_Time spent: 3.75h_

I decided to build a 18650 cell tester. I got this idea from my science fair project. I built a 3S34P battery pack from recycled 18650 cells. Testing each individual battery sucks and requires a tester. I purchased one from amazon but I want to learn how it works myself, so I decided to build one on my own. I decided on 4 channels because that's what I used and you could test one row of series at once. Each channel would measure voltage, capacity, and internal resistance. For sensing, I landed on the INA219. It measures both voltage and current over I2C, which keeps the wiring simple. One problem is that INA219s have the same default I2C address, so I'm also getting a TCA9548A I2C multiplexer to fix that issue. Each channel would also need a MOSFET and a load resistor to do controlled discharge. The capacity is measured by measuring current continuously during discharge and adding up (current x time) until the cell hits the voltage cutoff, which would be adjustable by the web app. Internal resistance would be measured at the very start by voltage sag. Apply load, measure how much voltage sags, and divide by the current. Results would go on an OLED screen and the WiFi dashboard that an ESP32 would host. Set up the GitHub repo and wrote some of the README. Did some research on the parts needed
